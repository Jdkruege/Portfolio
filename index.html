During our coursework we explored concepts such as: Grammars, Interpreters, Memory, Garbage Collection, Lambdas, Type checking, and variables. This class gave me an exceptional look into the mechanics that form languages I use every day.

Current Work

Senior Design - Software Engineering

My current focus is working on my senior design project. My project is the production of a wireless controller that can be used in freshman programming labs to engage students in programming. The idea behind this project is to engage students taking their first college class and make their first programming language fun. We have also been told this controller will be used in our customer's summer STEM program he runs with the local high school. I view this project as an opportunity to show off all the skills I have learned, while growing STEM interest in my community. I think this project is very important, as creating and retaining interest in STEM programs is necessary for our growing technological world.

Our system is built on a combination of C and C++ code, utilizing libraries for Bluetooth, SPI, and I2S. We are working on an Intel Edison, which is a Yocto Linux microcontroller.

I average 15 hours of work on this project a week.

What I have done:
  • Team Leadership
  • Timeline Scheduling
  • Documentation (Design document, Project plan, API)
  • Customer Relations
  • System Design
  • API Design
  • Extendable Class Design
  • Bluetooth Serial Communication
  • Hardware Evaluation
  • Software Testing
  • Hardware Testing

Through this project I have learned a lot about designing a system outside a classroom environment. In this project I do not have a grading standard or guide for how to make designs, so it comes down to evaluation of options and making choices that we, as a team, decide are best. We have had to make numerous decisions on this project regarding hardware components needed as well as software architecture.

When it comes to making APIs and extendable classes that students will need to use, we had to put a lot of consideration into them. We knew for a intro to programming course, requiring students to look at and create code that requires numerous lines to execute one task may be too stressful on them, so we decided we can abstract out all of that code into small, well defined API sets. This way students can call a method to read a sensor, or write to a sensor with no hassle. We also built extendable classes into our code, so students can build their own programs without knowledge of high-level concepts like threads.
